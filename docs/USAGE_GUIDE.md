# 売上ダッシュボード 使い方ガイド

> **Demo Changes**: このガイドはオリジナルのダッシュボード UI/UX を説明しています。デモ版では全データがシミュレーションです。

本ドキュメントは、各タブ・各ペインで「何をしたら何が判るのか」を簡潔にまとめたものです。仕組み（データの作り方）は `DASHBOARD_GUIDE.md` を参照してください。

- **何をする** … 操作（見るだけも含む）
- **判ること** … その操作で得られる情報

各ペイン内の「算出方法」展開ノートは、数字の作り方（保険枠の推定式・居住者の定義など）を示す別物です。本ガイドは「使い方」に特化しています。

---

## 1. 全体の使い方（ヘッダー）

ダッシュボードは画面上部のヘッダーで期間と表示を切り替え、その下の 7 タブ（拠点 / 分析 / サービス / 利用者 / 推移 / 拠点（配賦）/ 拠点（配賦2））で内容を見る構成です。

### 1.1 期間セレクタ（開始月 〜 終了月）
- **何をする**: ヘッダー左の 2 つのプルダウンで開始月・終了月を選ぶ。
- **判ること**: 選んだ期間の集計が、全タブに共通で適用される。開始月 > 終了月になると自動で補正される。単月を見たい時は開始月と終了月を同じにする。

### 1.2 About リンク
- **何をする**: クリックで仕組み解説ページへ遷移。
- **判ること**: データパイプライン（毎月 1 日の自動更新）、主要数値、売上の拠点帰属ルール、拠点詳細モーダル内の 2 つの切り口（当拠点で請求があった利用者 / 居住者の全サービス請求）。

### 1.3 ダークモード切替
- **何をする**: 月アイコンをクリック。
- **判ること**: ブラウザに保存されるので次回以降も維持される。

### 1.4 データ更新ボタン（管理者のみ）
- **何をする**: ログイン中の管理者にのみ表示。クリックで確認ダイアログ → スナップショット再生成（約 2 分）。
- **判ること**: 押下後 30 分はクールダウン（連打防止）。完了後はブラウザで再読み込みすると反映される。

### 1.5 ログアウト
- 右上のログアウトリンクでセッション終了。

### 1.6 基本の流れ
1. ヘッダーで**期間を選ぶ**
2. 知りたい観点に応じて**タブを切り替える**
3. 各ペインを**必要に応じて並び替え・絞り込み・ドリルダウン**

### 1.7 タブの使い分け早見表

| 知りたいこと | 見るタブ |
|---|---|
| 拠点ごとの売上規模・構成・推移 | **拠点** |
| 保険利用率、未使用枠、営業機会 | **分析** |
| サービス種別ごとの売上・利用者数 | **サービス** |
| 個別の利用者の明細 | **利用者** |
| 全社・カテゴリ・拠点の月次推移 | **推移** |
| 拠点ごとの配賦込み売上・スタッフ稼働状況 | **拠点（配賦）** |
| 配賦込み + 管理会計手入力の収入を含めた拠点比較 | **拠点（配賦2）** |

### 1.8 ヘッダー直下のグローバル KPI（全タブ共通）
ヘッダー下には期間合計の KPI カードが常に並び、タブを切り替えても同じものが見えます。

- **合計売上**: 期間の売上合計。サブ行は「平均/人」と「平均/拠点」。
- **保険負担額**: 保険制度負担分の合計。サブ行は全体に占める比率。
- **自己負担額**: 利用者自己負担の合計。サブ行は全体に占める比率。
- **利用者数**: 期間に何らかの売上があった利用者の延べ数。サブ行に拠点数と新規/終了の純増。
- **非売上**: 現在無効（※「FAQ Q5」参照）。

### 1.9 AI 分析ボタン（全 16 scope に配置済み）
- **何をする**: 紫グラデーションの「AI 分析」ボタンをクリック。ボタン直下にインラインパネルがスライドダウンして観察ポイント 3-5 項が箇条書きで表示される。
- **判ること**: その画面で表示しているデータ（期間・カテゴリ chip・検索・行 mute・拠点比較の選択拠点まで全部反映）を OpenAI gpt-5-mini が解釈し、異常値・前月比・偏り・経営アクション提案を生成する。**画面の数字を読み上げるだけではなく洞察を返す**。
- **scope の概念**: ボタンが置かれた場所ごとに AI が見る範囲が変わる。例えば拠点詳細モーダルの AI ボタンはその拠点 1 件分のみ。利用者モーダルはその利用者 1 件のみ。データはその場で sanitize（内部 ID 等を除去）してから送られる。
- **ストリーミング**: 数秒で最初の bullet が出始め、徐々に追加される。完了まで合計 5-30 秒程度。
- **フィルタとの連動**: chip / 検索 / 行 mute / 選択拠点 / モード切替 — どれを動かしても **その時点で開いている AI パネルは破棄される**。次回クリックは新しい状態で再分析。古い結果が残ることはない。
- **適用フィルタ**: パネル末尾の meta 行に「期間 / 拠点名 / カテゴリ / 検索 / 選択拠点 N 件 / 除外 N 項目」が表示される。**画面で何を絞ったかが透けて見える**。
- **キャッシュ**: 同じ scope + 同じフィルタなら 5 分以内は即座に同じ結果（meta に `cache` 表記）。
- **レート制限**: 1 ユーザー 30 回/時で打ち止め（429）。
- **配置一覧**:
  - **ヘッダー** (`global`): 全社 KPI + 前月比アラート
  - **拠点タブ** (`tab:facilities`): chip / 検索反映
  - **拠点比較グラフ** (`tab:facilities-compare`): 選択拠点 + カテゴリ + 一人当たり + 絶対値/相対値モード反映
  - **分析タブ** (`tab:analysis`): 介護保険利用率・営業機会
  - **サービスタブ** (`tab:services`): カテゴリ chip + 並び替え反映
  - **利用者タブ** (`tab:persons`): 検索クエリ反映、上位 50 名 + 統計サマリで送信
  - **推移タブ** (`tab:history`): 表示モード（カテゴリ/拠点）+ アクティブカテゴリ反映
  - **拠点（配賦）/ 拠点（配賦2）**: chip 反映
  - **drill-down モーダル** (`modal:facility / card-detail / card-detail2 / person / care-level / service-users / fac-gap`): 各エンティティ単位

### 1.10 前月比アラートバー
- **何をする**: 「詳細 ▾」で展開。単月・期間いずれでも表示される。
- **判ること**: 先頭のピル（▲施設増収 / ▼施設減収 / 新規 / 終了）は**期間終了月**の前月比サマリー（単月選択時は当月）。期間選択時はピル右側に対象月が注記される。展開すると施設ごとの前月 → 今月 → 変化率が出る。期間選択時は**月ごとのセクションに分かれ、各月の前月比が時系列順に並ぶ**。

## 2. 拠点タブ

**目的**: 拠点別の売上規模・業態構成・月次推移を俯瞰する。

**こういう時に見る**: 「どの拠点が稼いでいるか」「どの拠点が伸び / 縮んでいるか」「業態別の構成はどうか」。

### 2.1 検索ボックス＋CSV＋件数バッジ（ツールバー）
- **何をする**: 「拠点名で絞り込み…」に部分一致入力でテーブル・グラフを絞る。「CSV」で現在の一覧を書き出す。バッジは該当件数。
- **判ること**: 特定拠点だけを素早く絞り込める。CSV は画面上のフィルタ・単位を反映する。

### 2.2 カテゴリチップ（種別）
- **何をする**: 「全て」「有料老人ホーム」「訪問介護」などを**単一選択**でクリック。同じチップを再度押すと「全て」に戻る。
- **判ること**: その業態だけの拠点で、グラフと表が絞り込まれる。上部の KPI カードはこのフィルタの影響を受けず、常に期間合計を示す。

### 2.3 単位切替（億 / 百万 / 万 / 千）
- **何をする**: チップをクリックで単位切替。初期値は「百万」。
- **判ること**: グラフと表の金額表示が指定単位に揃う（KPI カードの合計値は ¥ のまま）。CSV にも反映。

### 2.4 拠点グラフ（横スクロール・色分け棒）
- **何をする**: 横にスクロールして全拠点を眺める。棒をクリックで拠点詳細モーダルを開く。
- **判ること**: 拠点ごとの売上規模の順位と、業態構成（色）の内訳。スクロールに追随して Y 軸目盛が自動調整され、画面内に見えている棒が常に適切な高さで収まる。

### 2.5 拠点テーブル
- **何をする**: 各行は拠点。行クリックで**拠点詳細モーダル**が開く。
- **判ること**: 拠点ごとの「保険負担額 / 自己負担額 / 合計 / 6 ヶ月スパークライン推移」（単月選択時）、または「合計 / 推移 / 前月 / 変化率」（期間選択時）。表示順は合計降順で固定（列クリックソートは不可）。

### 2.6 拠点比較グラフ
期間（複数月）を選択したときに表示されるセクション。重ねて推移を比較できる。

- **絶対値 / 相対値 (開始月=100) 切替**: 絶対値は金額そのまま、相対値は各拠点の開始月を 100 とした指数。**成長率で比べたい時は相対値**。
- **一人当たり**: 居住施設（住宅型・サ高住・GH 等）でのみ有効。居住者数で割って正規化する。
- **ピッカー**: 1 行目は「全体」＋「リセット／全クリア」、以降はカテゴリごとにその配下の拠点チップ。複数選択可。
- **単月選択時の挙動**: 空ではなく「直近 6 ヶ月の推移を比較」に切り替わり、期間選択と同じようにグラフが出る。
- 当期間に売上がある拠点が 0 件の場合は空状態のヒントが出る。

### 2.7 拠点詳細モーダル（テーブル/グラフクリック後）
- **請求内訳**: カテゴリ → サービス種別ごとの保険 / 自己 / 合計。カテゴリチップが上に出た時はその絞り込み下での内訳。
- **推移**: 6 ヶ月または選択期間の当拠点売上推移。
- **居住者の全サービス請求**: その拠点に住んでいる利用者が**他拠点も含めて**当月に使った全サービスの合計（居住者ベース）。
- **当拠点で請求があった利用者**: 住んでいるかどうかに関わらず**当拠点で請求が立った**利用者（通所利用者・外部利用者を含み、当拠点分のみ）。ソート可。
- **CSV**: 請求内訳を書き出し。
- 両ビューの違いは FAQ Q6 も参照。

## 3. 分析タブ

**目的**: 保険の利用率と未使用枠、営業機会（クロスセル）を把握する。

**こういう時に見る**: 「保険枠がまだ使える居住者は誰か」「単一サービスしか使っていない人を洗い出したい」「拠点ごとの利用率を見たい」。

※ 介護度・利用率はデータベースに介護度データが無いため、サービス種別と金額からの**推定値**です（本ガイド内でも都度明示）。

### 3.1 居宅系入居者 — 保険利用状況 KPI
- **保険未使用枠（推定）**: 期間内の住宅型・サ高住入居者の、推定介護度別上限 × 月数から保険請求実績を引いた残り枠の合計。
- **保険利用 入居者**: 保険制度の対象サービスを 1 件以上使った居住者数（自費のみの人を除く）。
- **低利用率（50%未満）**: 推定上限の半分以上が未使用の居住者数。

※ KPI カードはいずれも数字表示のみ（クリックで詳細は開かない）。

### 3.2 介護度分布ドーナツ（推定）
- **何をする**: セグメントをクリック → その介護度の利用者一覧モーダルが開く。
- **判ること**: 居住者の推定介護度分布と、サマリー（対象人数 / 平均利用率 / 未使用枠合計）。モーダルではソート可、行クリックで利用者詳細へ。

### 3.3 居宅系施設（住宅型有料・サ高住）— 支給限度額の利用分析
- **何をする**: 列ヘッダークリックで昇順/降順にソート。
- **判ること**: 施設 / 入居者 / 売上 / 平均/人 / 保険利用 / 平均利用率 / 低利用率 / 未使用枠 / 平均未使用枠 / 自費のみ / 検討中。**どの施設に未使用枠が多く眠っているか**を見るのが基本用途。

### 3.4 施設系（障がい者GH・介護付有料・認知症GH・特養）
- **何をする**: 列ソート可。行クリックで拠点詳細モーダル。
- **判ること**: 施設 / 入居者 / 売上 / 平均/人 / 検討中。限度額分析の対象外施設の売上概況のみ。

### 3.5 サービス拠点（訪問系・通所・就労・相談等）
- **何をする**: 列ソート可。行クリックで拠点詳細モーダル。
- **判ること**: 拠点 / 利用者 / 売上 / 平均/人 / 保険利用 / 平均利用率 / 低利用率。**他社サービス利用分は不明のため参考値**（セクションタイトル注記）。自社居住者は上の居宅系セクションでカウント済み、ここには含まれない。

### 3.6 クロスセル機会 KPI
- **単一サービス利用者**: 住宅型・サ高住入居者のうち、1 サービスしか使っていない人数。
- **訪問介護未利用**: 居住者のうち訪問介護を使っていない人数。
- **訪問看護未利用**: 同様に訪問看護未利用（要主治医指示書の注記あり）。

### 3.7 クロスセル詳細テーブル
- **何をする**: 氏名・居住施設のみソート可。氏名セルをクリックで利用者詳細モーダル。最大 50 行まで表示。
- **判ること**: 単一サービス利用者の氏名 / 居住施設 / サービス数 / 未利用サービス。営業候補リストとして使える。

### 3.8 事業別売上構成
- **何をする**: 列ソート可。
- **判ること**: カテゴリ / 利用者数 / 売上 / 構成比 / 平均/人。**売上ミックスを俯瞰**するペイン。

### 3.9 施設別 前月比
- **何をする**: 見るのみ（ソート不可）。
- **判ること**: 施設ごとの前月 / 今月 / 変化率。**急増・急減の検知**に使う。

### 3.10 現在無効なセクション
以下は `csm`（customer_service_map）テーブルおよび `billing_records_data` の削除に伴い、データが取得できず無効化中です（画面には「※ 現在無効」と表示）。

- **パイプライン（検討中）**
- **入居者 契約期間分布**
- **KPI「非売上」カード**（ヘッダー下の 5 番目）

## 4. サービスタブ

**目的**: サービス種別（通所介護・訪問介護・福祉用具等）ごとの売上と利用者数を見る。

**こういう時に見る**: 「どのサービスが主力か」「単価の高いサービスは何か」「特定業態内のサービス別売上」。

### 4.1 カテゴリチップ
- **何をする**: 単一選択。クリックでそのカテゴリ内のサービス種別だけに絞り込む。
- **判ること**: 業態内の内訳を切り出せる。

### 4.2 ソート切替（合計順 / 平均順）
- **何をする**: 「合計順」は売上合計の降順、「平均順」は利用者一人あたり売上の降順。
- **判ること**: 「合計順」で主力サービス、「平均順」で高単価サービスを見分けられる。

### 4.3 サービスグラフ（棒）
- **何をする**: 棒をクリックでそのカテゴリに絞り込む（再度押すと全体に戻る）。
- **判ること**: サービス種別ごとの売上規模。

### 4.4 サービステーブル
- **何をする**: 見るのみ（列クリックソート不可）。並び順はソート切替に従う。
- **判ること**: サービス種別 / 利用者数 / 売上合計 / 平均/人。1 人が複数サービスを使うと各サービスでカウントされる点に注意（method-note 参照）。

## 5. 利用者タブ

**目的**: 個別の利用者を検索し、サービス利用・請求明細を確認する。

**こういう時に見る**: 「特定の人が何に使っているか」「どの施設の人か」「契約サービスと月次請求の詳細」。

### 5.1 検索ボックス（氏名・ID・施設名）
- **何をする**: 入力するたびに即時フィルタ（ライブ検索）。氏名・person_id・居住施設の部分一致。
- **判ること**: 対象者を即座に絞り込める。

### 5.2 利用者テーブル
- **何をする**: 列ヘッダークリックで昇順/降順にソート。ページング可能。行クリックで**利用者詳細モーダル**。
- **判ること**: 氏名 / 居住施設 / サービス数 / 保険負担額 / 自己負担額 / 合計。退会フィルタは既に廃止されており、当月売上がゼロの人は自然と除外される。

### 5.3 利用者詳細モーダル（行クリック後）
- **基本情報**: 氏名 / ID / 居住施設
- **契約サービス**: 契約中のサービス一覧（開始・終了日含む）
- **支給限度額の利用分析（推定）**: 介護度推定値 / 上限 / 保険請求額 / 利用率 / 未使用枠（ドーナツ＋表）
- **月次請求明細**: カテゴリ / サービス種別 / 請求施設 / 保険 / 自己 / 合計
- **CSV**: 利用者情報 + 全請求明細を書き出し

## 6. 推移タブ

**目的**: 月次売上の推移を把握する。

**こういう時に見る**: 「全社トレンドを見たい」「カテゴリ別 / 拠点別に伸び縮みを比較したい」。

### 6.1 モード切替（カテゴリ別 / 拠点別）
- **何をする**: セグメントボタンで切替。
- **判ること**: カテゴリ別は積み上げエリアチャートで全社構成を時系列に。拠点別は拠点ごとの線グラフ。

### 6.2 カテゴリチップ（カテゴリ別モード時のみ）
- **何をする**: チップのオン/オフでそのカテゴリを表示/非表示。
- **判ること**: 特定カテゴリだけを強調して比較できる。

### 6.3 推移グラフ
- **何をする**: ホバーで月ごとの値をツールチップ表示。
- **判ること**: 選択した期間の月次変化。請求行数が極端に少ない月は不完全データとして除外されている（method-note 参照）。

## 7. 拠点（配賦）タブ

**目的**: 拠点単位で売上 + 社内間配賦 (Google Sheets「配賦マトリクス」由来) を統合表示し、配賦込み合計でランキング・推移を見る。**スタッフ稼働の実態が居住拠点側に反映された数字を観察するため**のタブ。

**こういう時に見る**: 「居住拠点のスタッフが訪問業務でどれくらい稼働しているか」「ステーション単独の billing と配賦後の実質収益で差がどれくらいあるか」「配賦込みでの拠点ランキング」。

データソースは `journal_entries`（仕訳明細）+ `Google Sheets「配賦マトリクス」`。既存の拠点タブ（§2）が `billing_records` 由来なのと違い、こちらは仕訳粒度で集計し、さくら拠点は `さくら1` / `さくら2` に split される。

### 7.1 検索ボックス + AI 分析ボタン
- **何をする**: 「拠点名で絞り込み…」で部分一致絞り込み。右端の「AI 分析」ボタンで scope=tab:card-view の AI 解析が走る（§1.9 参照）。
- **判ること**: 特定拠点だけを素早く絞り込める。AI 分析は配賦込みデータ全体の観察を返す。

### 7.2 表示モード（カード / グラフ）
- **何をする**: 「カード」「グラフ」の seg ボタンで切替。
- **判ること**:
  - **カード**: 各拠点が 1 枚のカード。「売上」セクション（自拠点 native）+ 「社内間配賦」セクション（流入は緑+、流出は赤−）+ 配賦込み合計を表示。
  - **グラフ**: 横軸=拠点、縦軸=配賦込み合計、棒の色分け=サービスカテゴリ。配賦行も該当カテゴリに反映される（例: 訪問介護（鹿）の流入は訪問介護カテゴリに加算）。

### 7.3 並び替え（売上順 / 名前順）
- **何をする**: seg ボタンで切替。
- **判ること**: 売上順は **配賦込み合計** で降順ソート。名前順は 50 音。

### 7.4 カテゴリチップ（種別）
- **何をする**: 単一選択。クリックで配賦込みのカテゴリ別表示に絞り込む。
- **判ること**: その種別のサービスを native + 配賦の両方で持つ拠点だけが残り、棒のセグメントもその種別だけ表示される。

### 7.5 拠点詳細モーダル（カード/棒クリック）

クリックすると詳細モーダルが開く。以下の構成:

- **上部 summary**: 配賦込み合計 / 売上 / 社内間配賦（流入・流出別、符号付き）/ 保険負担額 / 自己負担額。**カテゴリ chip / 行 mute / 配賦行 mute** に応じて即時再計算。
- **推移チャート**: 各サービスの折れ線（カテゴリ系統色）+ 「合計」(配賦込み)の点線。配賦行を mute すると合計線がリアルタイムでスケールダウンする。
- **請求内訳テーブル**: カテゴリ・サービス・請求元（個人 / 自治体）+ 保険・自己・合計。クリックで mute、再クリックで復活。
- **社内間配賦テーブル**: 相手先 + 金額。**行クリックで mute** → 上部 summary・チャートの合計線が即座に更新。
- **当拠点で請求があった利用者**: その拠点で billing が立った利用者一覧（行クリックで利用者モーダルへ）。
- **AI 分析ボタン**: モーダル header にあり、開いている拠点 + 期間 + 適用中の mute 全部を加味して scope=modal:card-detail を解析。

### 7.6 既存「拠点」タブとの違い

| 観点 | 拠点タブ (§2) | 拠点（配賦）タブ (§7) |
|---|---|---|
| データソース | `billing_records` (月次ロールアップ) | `journal_entries` (仕訳明細) + Google Sheets 配賦 |
| さくら拠点 | `さくら拠点` 1 件 | `さくら1` / `さくら2` に split |
| 配賦の扱い | 反映なし（billing 名義のまま） | 流入/流出を表示・合計に反映 |
| 食事代等 facility-level 明細 | 一部表示なし | すべて表示 |
| 主用途 | 拠点別の業態・売上規模 | 配賦込み拠点比較・スタッフ稼働 |

両タブは目的が違うため**併存**しています。経営者向けの俯瞰は §2 の拠点タブ、運営側の人員配置・稼働分析は §7 の拠点（配賦）タブが向いています。

## 10. FAQ

### Q1. データはいつの時点？どう更新される？
A. データは**月次のスナップショット**として凍結されています。ヘッダーの「データ更新」ボタン（管理者のみ表示）で最新のスナップショットを再生成できます（約 2 分）。再生成後は押下者のブラウザで 30 分間ボタンが無効化されます（連打防止）。

### Q2. 「保険未使用枠（推定）」は何を計算している？
A. 各居住者の**推定介護度**から導いた月額上限と、実際の保険請求額との差を、期間内で合計した値です。介護度は DB に格納されておらず、サービス種別と金額からの**逆算推定**のため、処遇改善加算などの影響で利用率が高めに出やすく、上限 100% キャップで表示しています。対象は居宅系施設の入居者のみ。

### Q3. 介護度は実データですか？
A. いいえ、**推定値**です。介護度データが本ダッシュボードの元データには含まれていないため、利用額とサービス種別から逆算しています。参考値として扱ってください。正確な介護度が必要な場合は元の介護記録を参照してください。

### Q4. 拠点比較グラフで単月を選ぶとどうなる？
A. 単月選択時は「直近 6 ヶ月の推移を比較」モードに自動切替され、選んだ月を含む過去 6 ヶ月が表示されます。空白にはなりません。当期間に売上がある拠点が 0 件の場合のみ、空状態のヒントメッセージが出ます。

### Q5. ヘッダー右端の「非売上」カードが無効なのはなぜ？
A. 元データ `billing_records_data` テーブルが削除されたため、現在無効化されています。復旧予定は未定。同様に分析タブの「パイプライン（検討中）」「入居者 契約期間分布」も `csm` テーブルの削除に伴い無効です。

### Q6. 「居住者の全サービス請求」と「当拠点で請求があった利用者」の違いは？
A. 視点が違います。
- **居住者の全サービス請求**: その拠点に**住んでいる人**の当月請求をすべて合算（他拠点で受けたサービスも含む）。住まいを軸にした「当施設居住者の全体像」。
- **当拠点で請求があった利用者**: **当拠点で請求が立った人**の、当拠点分のみの請求。居住者に限らず通所利用者や外部利用者も含む。場所を軸にした「当施設の売上明細」。

### Q7. CSV はどこから出せますか？
A. 以下の場所に「CSV」ボタンがあります。
- 拠点タブ / サービスタブ / 利用者タブ / 分析タブ のツールバー（画面上の絞り込み・並び替え・単位が反映される）
- 拠点詳細モーダル（請求内訳を出力）
- 利用者詳細モーダル（利用者情報 + 全請求明細を出力）

### Q8. 特定月だけ見たい時は？
A. ヘッダーの開始月と終了月を**同じ月**に設定してください。単月モードでは前月比アラートバーが当月の前月比を表示し、拠点比較グラフは「直近 6 ヶ月」モードに切り替わります。

### Q9. ダークモードに切り替えたい
A. ヘッダー右側の月アイコンをクリック。ブラウザに設定が保存され次回以降も維持されます。

### Q10. 拠点比較の「一人当たり」表示が効かないのはなぜ？
A. **居住施設（住宅型・サ高住・GH 等）でのみ有効**だからです。訪問介護・通所などサービス拠点では居住者の概念が無いため、一人当たり換算は適用されません。チェックを入れても居住施設以外の線は変化しません。

### Q11. 「検討中」とは何を指す？
A. 過去は `csm`（customer_service_map）テーブルの契約ステータスから「検討中（入居候補）」を集計していましたが、**現在は該当テーブルが削除されておりデータは空**です。表示枠だけが残っています。復活時期は未定。

### Q12. AppSheet の数字と若干合わないのはなぜ？
A. AppSheet と本ダッシュボードは**別経路で集計**しており、約 3.1% 程度のギャップが観察されています。主因は名寄せタイミングや再帰属ルール（訪問介護 3-pack・福祉用具の帰属先等）の違い。マネージャー報告用途では本ダッシュボードの数字を正とし、個別利用者の明細確認時は AppSheet を参照する運用が推奨されます。

### Q13. 「拠点（配賦）」と既存の「拠点」タブの違いは？
A. データソースと配賦の扱いが違います。詳細は §7.6 の対比表参照。
- 拠点タブ (§2): `billing_records` 月次ロールアップ。配賦は反映されず、billing 名義のままの集計。経営者向け俯瞰用。
- 拠点（配賦）タブ (§7): `journal_entries` 仕訳明細 + 社内間配賦。スタッフ稼働実態を反映した拠点比較。運営側分析用。
両タブの数字は config.py の name 解決ルールにより最大 0.2% 程度ずれることがあります。さくら拠点は `journal_entries` 系では `さくら1` / `さくら2` に split されます。

### Q14. AI 分析ボタンは何を見ているか？
A. ボタンが置かれた場所（scope）の現在の表示状態を、サーバー側で sanitize（内部 ID 等を除去）してから OpenAI gpt-5-mini に送信し、観察ポイントを箇条書きで生成しています。全 16 scope（ヘッダー / 全 7 タブ / 拠点比較グラフ / 7 種の drill-down モーダル）に配置済み。

scope と適用フィルタの主な対応:
- `global` (ヘッダー): 期間内全月の全社 KPI + 前月比アラート
- `tab:facilities`: chip（多選択）+ 検索が反映
- `tab:facilities-compare` (拠点比較グラフ): 選択拠点 + カテゴリ + 一人当たり + 絶対値/相対値モード
- `tab:services`: カテゴリ chip + 並び替えモード
- `tab:persons`: 検索クエリで絞り込み済みの利用者上位 50 + 統計サマリ
- `tab:history`: 表示モード（カテゴリ/拠点）+ アクティブカテゴリ
- `tab:card-view / card-view2`: chip（多選択）
- `modal:*`: 各エンティティ 1 件 × 期間 × mute 状態

**フィルタ変更との連動**: chip / 検索 / mute / 選択拠点 / モード切替 — どれを動かしても **その時点で開いている AI パネルは破棄**されます。次回クリックで新しい状態を再分析。古い結果が画面に残ることはありません。

パネル末尾の meta 行に「期間 / 拠点名 / カテゴリ / 検索 / 選択拠点 N 件 / 除外項目数」が表示されます。データは GCP 内で完結し外部に保存されません（cache は短期 in-memory のみ）。

### Q15. 配賦行（流入/流出）を mute すると何が変わるか？
A. 拠点詳細モーダル（§7.5）の社内間配賦テーブルで行をクリックすると、その配賦行が打ち消し線になり**合計と推移チャートの「合計」点線がリアルタイムで再計算**されます。例えば「中央中央からの流入 +¥510K」を mute すると、その金額が合計から差し引かれ、推移チャートの過去全月でも同じ counterpart-code の流入が除外されます（時系列を通した整合性のため）。再クリックで復活。

## 付録 A: 用語

| 用語 | 意味 |
|---|---|
| **居住者 / 入居者** | 当社の居住型施設（住宅型・サ高住・GH 等）に住んでいる利用者 |
| **居宅系施設** | 住宅型有料老人ホーム・サ高住（支給限度額分析の対象） |
| **施設系** | 障がい者GH・介護付有料・認知症GH・特養（支給限度額分析の対象外）|
| **サービス拠点** | 訪問系・通所・就労・相談等の拠点（居住者ではなくサービス利用者を対象）|
| **支給限度額** | 介護保険制度で介護度ごとに設定された月額上限 |
| **利用率** | 保険請求実績 ÷ 推定上限（%） |
| **未使用枠** | 推定上限 − 保険請求実績（¥） |
| **低利用率** | 利用率が 50% 未満 |
| **自費のみ** | 保険制度対象サービスを使わず自己負担サービスのみの居住者 |
| **検討中** | 入居候補（※ 現在無効） |
| **新規 / 終了** | 前月比で初めて / 最後に売上が立った利用者数 |
| **前月比** | ある月と前月の差分。アラートバーは単月選択時は当月の前月比、期間選択時は期間終了月の前月比をピル表示し、詳細展開で期間内の各月の前月比を時系列で表示 |
| **社内間配賦 / haifu** | ステーション (訪問介護/看護事業所) が billed した売上の一部を、実際にケアした居住拠点側へ移し替える月次の社内仕訳。`配賦マトリクス` (Google Sheets) で管理 |
| **流入** | 配賦の受取側。居住拠点に対するプラス値。自拠点スタッフが他拠点向けに行った訪問業務の対価 |
| **流出** | 配賦の払出側。ステーションに対するマイナス値。billed したが他拠点スタッフ実働分を引き渡した金額 |
| **居住拠点 / ステーション** | 当社の拠点を 2 種に大別。居住拠点 = 武・けやき・つつじ等の入居者居住型 (13 件)。ステーション = デモ介護福岡・中央・北・南、訪問看護福岡・中央・南・西 (8 件) |
| **配賦込み合計** | 自拠点 native 売上 + 流入 + 流出 (符号付き) の総和。スタッフ稼働を反映した実質収益 |
