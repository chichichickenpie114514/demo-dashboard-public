# Demo Sales Dashboard

Care facility revenue dashboard — demo/portfolio version with simulated data. Tracks billing, users, and service breakdowns across ~26 fictional facilities.

> **Demo Changes**: This is a sanitized demo of a production BI dashboard. All data is simulated (`generate_demo_data.py`, seed=42). Google OAuth accepts any Google account (no domain restriction). AI analysis uses DeepSeek API instead of OpenAI. No MySQL, AppSheet, or Google Sheets dependency — all data is pre-generated JSON in `data/`. See [docs/DASHBOARD_GUIDE.md](docs/DASHBOARD_GUIDE.md) for the original production architecture.

## Architecture

```
Runtime (Flask, local or Cloud Run):
  User opens dashboard  →  Flask  →  local data/ in-memory cache  →  browser
                                     ↘  (optional GCS fallback)
                                     ↘  /api/ai-analysis (DeepSeek streaming)
```

No database at runtime. All API responses are pre-computed static JSON served from memory. In production, data was refreshed monthly via a Cloud Run Job pipeline (MySQL → SQLite → JSON → GCS); the demo skips this entirely.

## Features

- **拠点** — Revenue per facility, category chips, multi-color bars, dynamic Y-axis
- **分析** — Insurance utilization, estimated care level, unused-capacity analysis
- **サービス** — Revenue by service type with user counts
- **利用者** — Per-user billing detail with care-level inference and limit analysis
- **推移** — Monthly trends by category and facility (12 months: 2025-04 ~ 2026-03)
- **拠点（配賦）** — Per-facility card view with internal allocation (haifu) overlay; card and chart modes; per-row mute reflected in chart
- **拠点（配賦2）** — 拠点（配賦） + management-accounting journal entries
- **AI 分析** — Scope-restricted streaming AI commentary across all tabs + drill-down modals (16 scopes). Every UI filter flows into the AI request
- **CSV export** — Tabs and modals export filtered/sorted views
- **Drill-down modals** — Facility / Card / Person details with mute-aware totals
- **Period range** — Select any start/end month; all KPIs, tables, and charts aggregate across the range

## Setup

```bash
pip install -r requirements.txt

# Generate simulated data (12 months, deterministic via seed=42)
python generate_demo_data.py

# Run locally
python server.py
# → http://localhost:5000 (Google OAuth — any account)
```

Required env vars (see `.env.example`):
- `GOOGLE_CLIENT_ID`, `GOOGLE_CLIENT_SECRET` — OAuth 2.0 credentials
- `SECRET_KEY` — Flask session signing (optional, auto-generated if not set)
- `DEEPSEEK_API_KEY` — AI analysis (optional)

## Deploy

```bash
bash deploy-next.sh  # Docker build → Cloud Run
```

Docker image contains the app code + `data/` directory with pre-generated JSON. No external data sources needed at runtime.

## Data

All data in `data/` is generated by `generate_demo_data.py`:
- 12 months (2025-04 through 2026-03)
- 194 JSON files: 16 per-month + 2 root-level
- Deterministic output (fixed `random.seed(42)`)
- ~26 fictional facilities, ~1800 synthetic persons
- Cross-file consistency: KPI totals ≈ sum of facility billing, person totals = sum of bills, etc.
